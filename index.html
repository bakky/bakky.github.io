《effective java》笔记（更新中） 2013-05-12


第21条：用函数对象表示策略


  • java没有函数指针，可以用对象引用实现同样的功能
  • 如果一个类仅仅导出一个方法，那么这个实例就等同于一个指向该方法的指针（例如常见的排序比较器使用：Comparator）这样就可以在某一个对象的方法中执行其它对象。

例：

Arrays.sort(stringArray,new Comparator<String>(){
     public int compare(String s1,String s2){
          return s1.length()-s2.length();
     }
})

每个比较器就是一个具体策略类，具体策略类往往使用匿名类声明（例如上面的例子），我们也可以声明一个普通的类实现Comparator接口。这个时候，我们可以考虑，因为用匿名类的话，每次执行调用的时候会生成一个新的对象实例，将这个函数对象存储在一个私有的静态final域中。

异常 2012-09-13

  • 异常都派生于Throwable类的一个实例。
  • Throwable下有两个子类分支：Error和Exception
  • Error类描述java运行时系统的内部错误和资源耗尽，这种情况很少出现
  • Exception下面又派生出两种异常分支：通常有程序导致的错误为RuntimeException，还有一类是IOException

java可以声明已检查异常，告诉编译器可能会发生什么样的错误。例如读文件方法可以throw FileNotFoundException异常。已检查异常可以抛出多个。

我们可以自定义一个派生于Exception或其子类的类

再次抛出异常或异常链：

这个可能大家在调用一些通用的模块的时候会发现它的异常信息是被包装了一层的，我们需要调用 对象的getCause()方法才能得到异常信息。

因为每个子系统可能会产生同样的异常，当我们的上层系统捕捉到这种异常时我们并不知道是哪个底层子系统抛出的。这个时候我们就可以将原是一场捕获后重新进行包装然后抛出代表这个子系统的异常，但是原始的异常信息也不会丢失。如：

try{
   access db
}catch(SQLException e){
   Throwable se = new MyAppException("db error");
    se.initCause(e);
    throw se;
}

代理、动态代理 2012-09-08


代理很容易理解，我们平时想要去买火车票，票贩子和代售点都可以理解为代理。 代理的结构图也很简单，代理类中含有真实类的引用，客户端调用代理类的方法，实际上代理类会去调用真实类的真实方法。那么代理类的主要作用是什么呢？就是在执行真实类的真实方法时，额外做些事情。

那么动态代理呢，试想我们买火车票、买飞机票用代理了，如果我们想要买国外奶粉，我们可以用代购，那么这个时候我们又要创建一个代理，如果有很多这种情况而且我们额外要做的事情差不多，那么创建很多代理类就浪费了，动态代理就是解决这种情况的，允许运行时指定其为实现任意一种接口实现类的代理类。

简单示例：
