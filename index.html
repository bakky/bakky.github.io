按照上面的教程一步一步来由于版本的问题还是会遇到一些问题的，这里作个记录：

  1. 下载最新版本的andriod studio(1.1.0),新建一个安卓工程,过程中会去下载andriod SDK,默认下载到/Users/${user}/Library/Android/sdk,也会去下载gradle,默认下载到/Applications/Android Studio.app/Contents/gradle/gradle-2.2.1目录下面,如果你已经在本地下载了gradle,你可以在Preferences->gradle中更改。
  2. 新建好第一个Hello World程序后，发现提示gradle sync failed,解决办法是下载jdk7(mac自带jdk6),同时设置JDK Location,这里如果你jdk7是采用安装包dmg形式安装的话，你要注意JAVA HOME路径是在/Library/Java/JavaVirtualMachines/jdk1.7.0_79.jdk/Contents/Home这里，而不是/System/Library/Frameworks/JavaVM.framework/,因为你用whereis java查看会发现它们都是软链到这个目录下的。
  3. 导入外部开源工程后，如上述教程中的9GAG,你会发现执行./gradlew build可能会失败，这是因为模块build.gradle文件中指定的compileSdkVersion和buildToolsVersion本地并没有，更改compileSdkVersion和buildToolsVersion成你本地版本即可。

对象排序 2013-07-27


提起java对象排序，大家首先会想到的是Comparator和Comparable,在《Thinking in Java》中是这样说明二者的区别的：

  • Comparable 是一个对象本身就已经支持自比较所需要实现的接口（如 String、Integer 自己就可以完成比较大小操作）
  • Comparator 是一个专用的比较器，当这个对象不支持自比较或者自比较函数不能满足你的要求时，你可以写一个比较器来完成两个对象之间大小的比较。

想象这么一个场景，商品对象Item有价格、销量、评分等属性，不同的业务需要根据不同的属性排序，大部分是根据商品销量排序，那么如何用这个排序比较器呢？

很容易想到，让商品这个类实现Comparable接口，在compareTo方法中比较商品销量，那么对于按销量排序的时候，只需要简单的利用Collections.sort(itemList)就可以达到效果,对于其它字段的排序，我们可以在需要的时候利用匿名类的方式，简化代码，如下：

Collections.sort(itemList, new Comparator<Item>() {
    @Override
    public int compare(Item o1, Item o2) {
        return o2.getPrice()-o1.getPrice();
    }
});

泛型相关知识整理 2013-07-17


  1. why generic programming?

    • 代码重用
    • 类型安全，编译时期发现错误

  2. what is generic programming?

    • 泛型分为泛型类和泛型方法。
    • 泛型处理几乎全在编译器中进行，Java编译器通过Code sharing方式为每个泛型类型创建唯一的字节码表示，并且将该泛型类型的实例都映射到这个唯一的字节码表示上。这就是通过擦除实现的

  3. why type erasue?

    • 将泛型类型实例关联到同一份字节码上。
    • 编译器只为泛型类型生成一份字节码，并将其实例关联到这份字节码上。
    • 避免浪费空间

  4. what is the type erasue?

    • 简单的理解为将泛型java代码转换为普通java代码，只不过编译器更直接点，将泛型java代码直接转换成普通java字节码。
    • 类型擦除的主要过程如下：
      1. 将所有的泛型参数用其最左边界（最顶级的父类型）类型替换。
      2. 移除所有的类型参数。
    • 对于泛型编译器还做了适当优化，会在编译期间对于插除后带来的问题尽可能的处理，例如通过桥接，插入相关字节码，保证正常执行，在进行翻译泛型变量的时候自动插入强制转换字节码

CountDownLatch与CyclicBarrier的使用和区别 2013-07-06


CountDownLatch


应用场景：有一个任务A想要往下执行，但必须要等到其他的任务B、C执行完毕后才可以继续往下执行。

使用方式：让任务A调用一个CountDownLatch对象的await()方法，其他的任务执行完自己的任务后调用同一个CountDownLatch对象上的countDown()方法，任务A将一直阻塞等待，直到这个CountDownLatch对象的计数值减到0为止。

那么CountDownLatch内部是如何实现让这个任务一直阻塞等待一直到所有其它任务执行完的呢？

粗略的想下，原理可能是初始化一个值为所有任务个数的原子计数器，当前线程一直循环等待，执行下去的条件是全局计数器值为0，其它任务线程执行完后全局计数器减1

来看看其源码：await()方法和countDown()方法

private final Sync sync;
public void await()
            throws InterruptedException {
    sync.acquireSharedInterruptibly(1);
}
public void countDown() {
    sync.releaseShared(1);
}

《effective java》笔记（更新中） 2013-05-12


第21条：用函数对象表示策略


  • java没有函数指针，可以用对象引用实现同样的功能
  • 如果一个类仅仅导出一个方法，那么这个实例就等同于一个指向该方法的指针（例如常见的排序比较器使用：Comparator）这样就可以在某一个对象的方法中执行其它对象。

例：

Arrays.sort(stringArray,new Comparator<String>(){
     public int compare(String s1,String s2){
          return s1.length()-s2.length();
     }
})

每个比较器就是一个具体策略类，具体策略类往往使用匿名类声明（例如上面的例子），我们也可以声明一个普通的类实现Comparator接口。这个时候，我们可以考虑，因为用匿名类的话，每次执行调用的时候会生成一个新的对象实例，将这个函数对象存储在一个私有的静态final域中。
